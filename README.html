First Steps

Now, to begin, you want to:

  1. Install the plugin: Copy CommandBook.jar into your Bukkit server's "plugins" directory. If the folder doesn't exist, create it.
  2. Change the configuration: Run your server once (with CommandBook) and then edit the config.yml file that is generated inside "plugins/CommandBook". Use /cmdbook reload in the console to reload the configuration.
  3. Setup permissions: Visit the CommandBook wiki to learn the permissions (and commands).

Gotcha: You need to have WorldEdit installed! It's used by CommandBook in a number of places. If you don't want to install the plugin, that's okay — just put WorldEdit.jar into your Bukkit root folder (outside of plugins/) and CommandBook will still be able use it.

Advanced Usage

CommandBook is pretty simple (/time dawn will do what you expect), but there is a lot more under the hood to discover. For example, there's no /teleport-to-coordinates command, but /tp takes something called a "location." Most other plugins simply assume that a location can only be a player name but CommandBook is smarter. A location can also mean the world's spawn location or a coordinate, so why limit yourself? Try these:

  • /tp 0,0,0
  • /tp #spawn

All commands that use a location all use the same location processing, so #spawn will work for every command that takes a location. Cool, right?

In a similar vein, commands that take a target have advanced target processing code, giving you the ability to use crazy tricks like /rocket #near to rocket everyone near you (yourself included)!
